# Implementation artifacts

This folder is the **BMAD output location for implementation-phase deliverables**. It stays separate from app code (e.g. `zaffr/`) so planning and tracking live in one place.

---

## What goes here (when using full BMAD workflows)

When you run the BMM **implementation** workflows, they read/write here:

| Artifact | Produced by | Purpose |
|----------|-------------|---------|
| **`sprint-status.yaml`** | Sprint Planning | Sprint plan and story status (what’s next, done, blocked). |
| **`{epic}-{story}-*.md`** or **`{story_key}.md`** | Create Story | One file per story; Dev Story and Validate Story use these. |
| **Story validation reports** | Validate Story | Output of story readiness checks. |
| **`tech-spec-*.md`** | Quick-spec (if used) | Technical specs and WIP (`tech-spec-wip.md`). |
| **`tests/test-summary.md`** | QA Automation Test | Summary of generated API/E2E tests. |
| **`epic-{n}-retro-{date}.md`** | Retrospective | Per-epic retrospective notes. |

So: **sprint status**, **story files**, **validation reports**, **tech specs**, **test summaries**, and **retrospectives** all belong in `_bmad-output/implementation-artifacts` when you use those BMAD commands.

---

## How this project is using it

Sprint Planning is run **before every sprint** (see Sprint Readiness Check). For the Zaffr todo app we’re following the **step-2 execution guide** and **epics-and-stories** in `_bmad-output/planning-artifacts/` and maintain `sprint-status.yaml` via Sprint Planning or manually. So:

- **Sprint and story tracking** is done in:
  - `docs/task-progress-documentation/step-2-build-execution-guide.md` (checkboxes per story)
  - `docs/task-progress-documentation/sprint-readiness-check.md`
- **Evidence and “what we did”** is in:
  - `docs/task-progress-documentation/documentation-of-my-bmad-driven-to-do-list-app.md`
  - `cursor-updates.md`

**`sprint-status.yaml`** in this folder is generated by **`/bmad-bmm-sprint-planning`** (or maintained manually) and is the single source of truth for story status. The Sprint Readiness Check and execution guide tell you to run Sprint Planning before every sprint. **Create Story** and **Dev Story** (if used) expect and create **story files** here.
